# Codebase Audit — 2026-04-26

> **Scope:** full codebase + docs (not just recent PRs). Twelve parallel audit agents covered:
> A1 sealed-mount, A2 core engine, A3 graph stack, A4 async/concurrency, A5 REST API,
> A6 VS Code extension, A7 versions + link rot, A8 docs vs reality, A9 cross-interface
> parity, A10 roadmap + TODOs, A11 test coverage gaps, A12 operational/prod-readiness.
>
> **Format:** every finding cites file path + line number, severity (P0 = data loss /
> security / corruption · P1 = user-facing breakage / wrong behavior · P2 = cleanup), and
> a proposed fix or "needs investigation". Missing-feature gaps include effort estimate
> (S = <1 day · M = 1-3 days · L = >3 days).

## Executive summary — top items to fix first

1. **Atomic-write gaps that can corrupt graph state** (P1) — `graph_rag.py:929`
   entity-graph `.msgpack` write is not atomic; `code_graph.py:34` json write is not
   atomic; `_gr_write_*_if_changed` lacks the Windows/cloud-sync fallback that
   `dynamic_graph_backend.py:358-372` already implements. SIGINT mid-write zeros the
   files.
2. **`_persist_executor_internal` is never shut down** (P1, `graph_rag.py:131`) — leaks
   threads on every `switch_project`. Combined with `_executor.shutdown(wait=False)` at
   `main.py:617` silently abandoning in-flight community-rebuilds and graph-extractions,
   long-running servers leak threads + drop work.
3. **`governance.emit()` spawns one daemon thread per audit event** (P1,
   `governance.py:486`) — no pool, no flush, no join. Process shutdown kills daemons
   mid-`AuditStore.append()`, losing the exact data the audit log is supposed to
   guarantee.
4. **TOCTOU race on lazy lock initialisation** (P1, `query_router.py:65-75`,
   `graph_rag.py:43-46/50-55/90-93/118-134`) — two threads racing into the lock-creating
   `@property` each create a fresh lock; "locked" sections can run concurrently.
5. **`/ingest/refresh` uses MD5 to compare against SHA-256 hashes** (P1,
   `api_routes/ingest.py:121-123`) — re-ingests every file on every call, defeating
   dedup.
6. **`/query/stream` collects all chunks before yielding** (P1, `api_routes/query.py:160-182`)
   — defeats streaming entirely.
7. **`_extract_entities_gliner` calls `_ensure_gliner()` outside try/except** (P1,
   `graph_rag.py:3387`) — `ImportError` if `gliner` not installed crashes the whole
   query.
8. **GraphRAG `clear()` and `delete_documents()` mutate without `_graph_lock`** (P1,
   `graphrag_backend.py:104-118`) — concurrent `/graph/data` reads → `RuntimeError:
   dictionary changed size during iteration`.
9. **`tools/security.ts` is fully orphaned** (P1) — defines 5 LM tool classes + 5
   commands but is NOT imported by `extension.ts`. Sealed-mount Phase 6 has zero VS Code
   surface despite CLAUDE.md cross-interface parity rule.
10. **~25 `graph_rag_*` config knobs read via `getattr(self.config, "...", default)`
    but not declared on `AxonConfig`** (P1) — silently bypasses YAML loading + `/config`
    routes + the wizard. Per CLAUDE.md this is the explicit anti-pattern.
11. **`api.allow_origins` is dead code** (P1, `api.py`) — declared in `AxonConfig`,
    loaded from YAML, round-tripped on save, but no `CORSMiddleware` is ever
    instantiated. Operators set it expecting CORS; gets silent no-op.
12. **`_hard_revoke` crash window between staged-DEK promote and rotation-marker
    delete is unrecoverable** (P1, `share.py:902-910`) — the doc-comment claims "crash
    anywhere is recoverable" but this specific window is not.
13. **Hard revoke does NOT bump `version.json` `seq`** (P1, `share.py::_hard_revoke`)
    — grantees never trigger `refresh_mount`, hit `InvalidTag` instead of
    `MountSyncPendingError` (misleading failure surface).
14. **`SealedCache.create` walks `.security/` and copies wrap files into the temp
    cache** (P2, `cache.py:283-299`) — dek.wrapped + per-share wraps + `.sealed` marker
    end up in tempdir; downstream `is_project_sealed(cache.path)` would return True even
    on decrypted cache. Footgun.
15. **CHANGELOG.md missing v0.2.1 entry** (P1) — release shipped (PR #50) but no
    changelog row. CLAUDE.md still says "Current version: 0.2.0".

## All P1 issues (by area)

### Sealed-mount (A1)
- **`__init__.py:227-244`** — Four functions are still Phase-1 stubs after Phase 5
  shipped. `validate_received_sealed_shares` returns `[]`; `list_sealed_shares` returns
  `{"sharing": [], "shared": []}`; `resolve_owned_sealed_project_path` always raises;
  `project_rotate_keys` always raises `"not configured"`. `/share/list` silently returns
  no sealed shares; `/project/rotate-keys` always 404s. **Fix:** wire to
  `share.list_sealed_share_key_ids` and `revoke_sealed_share(rotate=True)`.
- **`share.py:902-910`** — `_hard_revoke` crash window between
  `os.replace(_rotation_dek_path, dek_wrap_path)` (line 906) and `_clear_rotation_context`
  (line 935) is unrecoverable on resume. **Fix:** make `_read_staged_dek` first try the
  live `dek.wrapped` if the staging file is gone, or write a "promoted" sentinel.
- **`share.py::_hard_revoke`** — does not bump `version.json` `seq`. **Fix:** call
  `axon.version_marker.emit(project_dir, ...)` at the end of `_hard_revoke` and
  `project_seal`.

### API + REST (A5)
- **`api_routes/ingest.py:121-123`** — `/ingest/refresh` uses **MD5** to compare against
  SHA-256-tracked sources. Comparison can never match → re-ingests every file on every
  call. **Fix:** replace `_hashlib.md5(...)` with `_compute_content_hash`.
- **`api_routes/query.py:160-182`** — `/query/stream` does
  `await asyncio.to_thread(lambda: list(brain.query_stream(...)))` BEFORE yielding —
  defeats streaming. **Fix:** queue + thread for true incremental delivery.
- **`api_routes/maintenance.py:118` vs `governance.py:250`** — Two
  `POST /project/maintenance` routes registered; whichever is included second silently
  shadows the first. **Fix:** consolidate to one handler.

### Graph + GraphRAG (A3)
- **`graph_rag.py:3387`** — `_extract_entities_gliner` calls `_ensure_gliner()` outside
  the try/except. ImportError crashes the query. Same for `_extract_relations_rebel`.
  **Fix:** wrap `_ensure_*` in the try, or `except (ImportError, Exception)`.
- **`graph_rag.py:929`** — `mp_path.write_bytes(raw)` for `.entity_graph.msgpack` is
  not atomic. **Fix:** route through `_gr_write_bytes_if_changed`.
- **`graph_rag.py:646-647, 676-677`** — atomic-write has no Windows/cloud-sync
  `PermissionError` fallback (the dynamic-graph backend at
  `dynamic_graph_backend.py:358-372` handles this). **Fix:** add copy+unlink fallback.
- **`code_graph.py:34`** — `path.write_text(json.dumps(self._code_graph), ...)` is not
  atomic. **Fix:** route through `_gr_write_json_if_changed`.
- **`graphrag_backend.py:96-98`** — `finalize()` swallows ALL exceptions silently.
  Broken Leiden returns "success". **Fix:** log and propagate.
- **`graphrag_backend.py:104-118`** — `clear()` + `delete_documents()` mutate
  `_entity_graph` without `_graph_lock`. **Fix:** wrap in `with self._brain._graph_lock`.
- **`graph_rag.py` — ~25 `graph_rag_*` config knobs** read via `getattr(self.config,
  "...", default)` but **not declared as `AxonConfig` fields**. Unreachable via YAML or
  `/config/set`. (Full list in A3+A8 sections; includes `graph_rag_global_answer_cache`,
  `graph_rag_extraction_cache`, `graph_rag_relation_shard_*`, etc.)

### Async / threading (A4)
- **`query_router.py:65-75`** + **`graph_rag.py:43-46/50-55/90-93/118-134`** — TOCTOU
  race on lazy lock initialisation. **Fix:** initialise eagerly in `AxonBrain.__init__`
  or guard with a class-level `_init_lock`.
- **`main.py:617`** — `_executor.shutdown(wait=False)` after `_flush_pending_saves()`
  abandons in-flight community-rebuilds + graph-extractions silently. **Fix:** track
  pending main-executor futures and `concurrent.futures.wait(timeout=...)` before
  shutdown.
- **`graph_rag.py:131`** — `_persist_executor_internal` is never shut down. Leaks
  threads on every `switch_project`. **Fix:** `shutdown(wait=True)` in
  `AxonBrain.close()`.
- **`governance.py:486`** — `emit()` spawns one new daemon thread per audit event.
  **Fix:** singleton `ThreadPoolExecutor(max_workers=1)` + `atexit` flush.
- **`security/cache.py:322-329`** — `SealedCache.wipe()` is not thread-safe; concurrent
  `close()` callers can both pass the `_wiped` check and race on directory removal.
  **Fix:** add `_wipe_lock = threading.Lock()`.

### Core engine (A2)
- **`vector_store.py:765, 791, 863`** — `MultiVectorStore.search`,
  `MultiVectorStore.batch_search`, `MultiBM25Retriever.search` instantiate
  `ThreadPoolExecutor(max_workers=min(4, len(self._stores)))`. With empty list →
  `ValueError`. **Fix:** early-return on empty.
- **`llm.py:395-412, 707-725`** + **`embeddings.py:138-152`** — Zero retry / backoff
  anywhere in `OpenLLM`. Ollama/OpenAI/Gemini/Grok/vLLM connect errors propagate raw.
  **Fix:** central retry decorator (`tenacity`) on transient errors; surface
  `LLMUnavailable` cleanly.
- **`loaders.py:469-484`** — `URLLoader._check_ssrf` runs once before the request and
  once after the final URL, but `httpx` follows redirects internally → intermediate hops
  not validated → SSRF via redirect to private IP. **Fix:** `follow_redirects=False`,
  loop checking `_check_ssrf` on each `Location`.
- **`loaders.py:490-494`** — `_MAX_FILE_BYTES` enforced AFTER `resp.text` fully buffers.
  10 GB response OOMs the worker. **Fix:** stream + check `Content-Length` upfront.
